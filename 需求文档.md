# 健康管家 — 需求规格说明书

## 1. 项目概述

### 1.1 项目背景

随着人口老龄化趋势加剧以及公众健康意识的提升，个人健康管理需求日益增长。大量慢性病患者（如高血压、糖尿病）需要长期服药并定期监测健康指标，传统的手动记录方式存在数据易丢失、提醒不及时、难以可视化分析等问题。因此，亟需一款便捷、智能、可视化的个人健康管理工具，帮助用户日常追踪用药情况、监测健康数据、形成健康报告。

### 1.2 项目目标

开发一款基于 Web 技术的移动端优先健康数据管理应用，面向个人用户，提供用药管理、健康指标监测、数据可视化、智能提醒等核心功能，实现健康数据的便捷录入、持久化存储、可视化分析与及时提醒。

### 1.3 目标用户

- 慢性病患者（需长期规律用药）
- 中老年健康关注人群
- 有健康管理需求的普通用户
- 需要辅助子女/父母管理健康数据的家庭用户

## 3. 非功能需求

### 3.1 性能需求

- 页面首次加载时间 < 3 秒（在4G网络环境下）
- 图表渲染流畅，无卡顿
- 数据操作（增删改）响应时间 < 300ms
- 应用支持离线查看已保存数据（所有数据 localStorage 本地存储）

### 3.2 兼容性需求

- 支持 Chrome 90+、Firefox 88+、Safari 14+、Edge 90+
- 移动端优先，适配 iOS Safari 和 Android Chrome
- 响应式布局，适配 320px ~ 428px 宽度的手机屏幕

### 3.3 安全需求

- 用户登录态通过 localStorage 管理，需防止 XSS 攻击导致 token 泄露
- 个人敏感信息（手机号、邮箱）本地存储，不上传服务器（当前为纯前端实现）
- 路由守卫确保未授权用户无法访问受保护页面

### 3.4 可靠性需求

- 数据持久化：所有用户数据写入 localStorage，刷新、关闭浏览器后数据不丢失
- 登录有效期：开发阶段设定3分钟，防止用户长期未操作后仍保持登录态
- 异常处理：localStorage 读取异常时自动回退到默认值，避免页面崩溃

### 3.5 用户体验需求

- 界面风格：清新绿色健康主题，符合医疗/健康类应用视觉规范
- 操作反馈：按钮点击、保存成功、删除确认等提供即时视觉反馈
- 动画效果：页面切换、弹窗出现、心率圆环等适度动画，不干扰操作
- 空状态处理：药品列表为空时展示空状态插图与提示文字

## 4. 数据需求

### 4.1 数据存储方案

当前版本为纯前端实现，所有数据存储于浏览器 localStorage：

| 存储键 | 数据内容 | 示例 |
|--------|---------|------|
| `token` | 登录令牌 | `"mock-token-123456789"` |
| `user` | 用户信息（用户名、姓名、头像） | `{"username":"admin","name":"张三"}` |
| `expiresAt` | 登录过期时间戳 | `"1234567890000"` |
| `health_profile` | 个人资料 | `{"name":"张三","gender":"男","age":30,"height":175,"weight":70}` |
| `avatarUrl` | 头像 DataURL | `"data:image/png;base64,..."` |
| `health_medicines` | 药品列表 | 数组，含名称、剂量、时间等 |
| `health_metrics` | 健康指标 | `{"heartRate":72,"bloodSugar":5.4,"sleepHours":7.2,"steps":6432}` |
| `theme` | 主题偏好 | `"dark"` / `"light"` |

### 4.2 数据同步需求

- 同一页面内不同组件通过 Pinia store 共享数据，实时同步
- 不同页面间通过 localStorage 实现数据持久化与跨页同步
- 刷新页面后，store 自动从 localStorage 恢复数据

## 5. 界面原型需求

### 5.1 页面结构

应用采用单页 AppShell 布局，底部固定 Tab 导航：

```
+----------------------------------+
|  [Logo] 健康管家    [通知] [头像]  |  ← 顶部导航栏
+----------------------------------+
|                                  |
|          主内容区域               |  ← 根据 Tab 切换页面
|                                  |
+----------------------------------+
| [首页] [药品] [数据] [我的]     |  ← 底部 Tab 导航
+----------------------------------+
```

### 5.2 核心页面说明

- **首页**：欢迎语 + 快捷功能 2x2 网格 + 今日用药提醒 + 健康数据卡片 + 血糖趋势图
- **药品页**：顶部搜索 + 药品列表 + 添加按钮 + 提醒设置
- **数据页**：实时心率 + 心率趋势图 + 睡眠时长图 + 健康报告
- **我的页**：头像 + 姓名 + 个人信息 + 设置 + 帮助中心

## 6. 技术约束

- 开发语言：JavaScript（不使用 TypeScript）
- 前端框架：Vue 3 + Composition API
- 构建工具：Vite
- UI 库：Ant Design Vue 4（部分自定义组件补充）
- 状态管理：Pinia 2
- 路由：Vue Router 4（history 模式）
- 图表：ECharts 5
- 图标：Font Awesome 6
- 纯前端实现，不依赖后端服务器

## 7. 验收标准

### 7.1 功能验收

- [ ] 用户可使用测试账号正常登录，3分钟后自动提示过期
- [ ] 首页健康数据与数据页、药品页数据实时一致
- [ ] 添加、编辑、删除药品后列表即时更新，刷新后数据保留
- [ ] BMI 根据个人信息中的身高体重动态计算
- [ ] 头像上传后显示圆形裁剪头像，刷新后保留
- [ ] 姓名编辑后保存，刷新后显示新姓名
- [ ] 深色模式切换后全局生效，刷新后保持主题
- [ ] 通知总开关关闭后所有提醒停止，开启后恢复
- [ ] 未登录用户访问任何内部页面自动跳转登录页

### 7.2 性能验收

- [ ] 页面加载时间 < 3 秒
- [ ] 图表渲染无明显卡顿
- [ ] localStorage 读写异常时页面不崩溃
